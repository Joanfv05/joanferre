Contexto

El proyecto

BatoiLOGIC es una empresa ficticia de distribución logística comarcal que necesitaba digitalizar todos sus procesos. El proyecto consiste en desarrollar un ecosistema completo: dos apps móviles Flutter, una API REST con Spring Boot desplegada en AWS EC2, un backoffice Windows Forms en C#, un backoffice Odoo con módulos personalizados y una app CLI de importación/exportación de datos en Java.

Mi rol

Arquitectura

Diseño de la arquitectura del sistema y modelo de datos relacional que soporta el ecosistema completo de cinco aplicaciones.

Backend

Desarrollo de la API REST completa con Spring Boot, Hibernate y documentación de endpoints para consumo desde todas las apps.

Mobile

Desarrollo de las dos apps móviles Flutter — cliente y repartidor — con geolocalización en tiempo real, estados de pedido y persistencia local SQLite.

Escritorio

Backoffice Windows Forms en C# con reconocimiento de voz, visualización del mapa de flota en tiempo real vía WebSocket y gestión completa de rutas.

ERP

Configuración de Odoo con módulos personalizados de repartidores y camiones, integración de albaranes y gestión de stock adaptada al flujo logístico.
